Introduction

Training Wheels is an open-source education platform that manages sandbox environments and training infrastructure for use in a classroom setting.

This product is in pre-alpha state. If you're a developer interested in working with us, please get in touch. Our current roadmap includes designing and finishing the user interface and moving the controller off Drupal and making it a stand-alone Symfony PHP app.

Is Training Wheels for me?

Does your training course require each student to have access to their own sandbox for hands-on application of the skills you're teaching? If it does, you could benefit from using Training Wheels.

For example, you may be teaching 20 students how to theme Drupal. You'd like each of them to have their own installation of Drupal available to work on, perhaps configured in a specific way with a few contributed modules installed. You probably also want a base theme setup for them to start working on.

A traditional approach to solving this problem is to require each student to bring their laptop with a LAMP stack pre-installed. You could distribute the base files and database to each student before the class, and require everyone arrive on the day with a working base site.

This option is problematic for many reasons:

  • What if your target audience (designers in this example) don't have the skills or time to precisely configure their local environments?
  • What if you need to change some settings in the base configuration at the last minute, requiring everyone to re-install?
  • What if a student falls behind or gets stuck in a certain place, can you bring their environment up to speed quickly so that they can continue with the course?
  • What if someone walks in on the day of training, can you quickly get them on board or would you have to turn them away?

The Training Wheels approach

Training Wheels solves this issue by configuring a single Linux server to be a multi-user developer environment, and then providing a controller site which allows an administrator command-line and web access to manage the student's environments.

You create a new cloud or physical server with your own domain name, for example mytraining.com. SSH into the machine and run the server configuration playbook. Run the controller setup playbook to install the controller. Everything is configured automatically and all configuration is open-source, meaning even the most pedantic sysadmins should be happy, as you can inspect and tweak every setting and configuration file in the extremely readable playbooks, before they are run on your server.

Flexibility

There are two components to Training Wheels: the controller site and the server setup playbooks.

The controller is an application built on Drupal, in the form of a distribution. It is designed to be extremely flexible in what kind of training course is being run. Do you need to provide each student an installation of Wordpress? Or a custom application? Or perhaps you only need a single Drupal site and each student should have a user account on that single site. The Training Wheels controller is designed from the ground up to handle these cases, although we are mostly developing for a LAMP stack.

You don't have to run the controller on the same server as the student environments if you don't want to - it supports administering the local server as well as a remote server over SSH.

The second component to an installation is the server configuration playbook, which is highly specific. We currently have a playbook to configure a server for a multi-user LAMP stack with SSH & FTP access for students, however we will surely accumulate many more playbooks in the future for setting up multi-user servers for other applications.

Who uses this?

We developed Training Wheels at Four Kitchens. It has been used to successfully deliver the following training courses:

  • Responsive Web Design: DrupalCon Denver 2012
  • Responsive Web Design: DrupalCon Munich 2012
  • PHP for non-programmers: BadCamp 2012
  • Drupal and node.js: BadCamp 2012
